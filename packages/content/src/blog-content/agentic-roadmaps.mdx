# Agentic roadmaps: building strategy that updates with real customer evidence

Traditional roadmaps are static documents that age the moment they're published. Customer priorities shift. Competitors launch features. Usage patterns change. By quarterly review, the roadmap no longer reflects what matters.

Agentic roadmaps are different. They incorporate customer evidence, competitive changes, and analytics signals continuously, creating living strategy documents that reflect current reality, not last quarter's assumptions.

## The Problem with Static Roadmaps

Static roadmaps fail in predictable ways:

### Assumption Decay

Roadmaps are built on assumptions about customer needs. Those assumptions decay:

```markdown
## Q1 Roadmap Assumption (October)
"Enterprise customers need SSO more than search improvements"

## Reality (January)
- 3 enterprise deals closed without SSO
- 52 VoC mentions of search frustration
- Competitor launched AI search
- SSO requests stable, search requests growing

The assumption was wrong, but the roadmap didn't update.
```

### Evidence Disconnect

Roadmap items often lose connection to their evidence:

```markdown
## Roadmap Item: Dashboard Redesign

**Original justification**: "Customers find the dashboard confusing"
**Evidence**: 3 support tickets from Q2 2025

**Current reality** (Q1 2026):
- Dashboard tickets down 80% after minor UX fixes
- No recent VoC mentions of dashboard issues
- Other pain points now more urgent

The item stayed on the roadmap after the problem was solved.
```

### Priority Drift

Priorities shift but roadmaps don't:

```markdown
## Priority Changes (Q4 → Q1)

| Item | Q4 Priority | Q1 Evidence | Should Be |
|------|-------------|-------------|-----------|
| SSO | P0 | Stable demand | P1 |
| Search | P2 | 3x increase in mentions | P0 |
| Mobile | P1 | Declining requests | P2 |
| API v2 | P3 | Enterprise deals blocked | P1 |

Static roadmap shows Q4 priorities.
Reality requires Q1 reprioritization.
```

## What Makes a Roadmap "Agentic"

An agentic roadmap has three properties:

### 1. Evidence-Linked Items

Every roadmap item connects to live evidence:

```markdown
## Roadmap Item: Search Date Filters

### Evidence Dashboard

**VoC Signal** (live)
- Zendesk mentions: 28 (↑ 12% this month)
- Gong mentions: 12 (↑ 8% this month)
- Community votes: 89 (↑ 15% this month)
- Sentiment: -0.6 (negative)

**Competitive Signal** (live)
- Competitor X: Has date filters ✅
- Competitor Y: Has date filters ✅
- Our position: Gap ❌

**Business Signal** (live)
- Deals mentioning this: 3 ($450K pipeline)
- Churn risk accounts: 2 ($195K ARR)
- Expansion blocked: 1 ($180K potential)

**Priority Score**: 8.7/10 (↑ from 7.2 last month)
```

## Building an Agentic Roadmap System

### Component 1: Evidence Aggregation

Connect roadmap items to evidence sources:

```typescript
interface RoadmapItem {
  id: string;
  title: string;
  description: string;
  status: 'proposed' | 'planned' | 'in_progress' | 'shipped';
  targetQuarter: string;
  
  // Evidence connections
  evidence: {
    vocThemes: string[];        // Links to VoC theme IDs
    competitorGaps: string[];   // Links to competitive gaps
    supportTickets: string[];   // Zendesk ticket IDs
    gongMentions: string[];     // Gong insight IDs
    communityPosts: string[];   // Community post IDs
    blockedDeals: string[];     // CRM opportunity IDs
    analyticsSignals: string[]; // Analytics insight IDs
  };
  
  // Calculated fields
  priorityScore: number;
  lastScoreUpdate: Date;
  scoreHistory: { date: Date; score: number }[];
}
```

Process incoming signals and update roadmap:

### Component 3: Priority Scoring

Calculate and track priority scores:

```typescript
interface PriorityFactors {
  // VoC factors
  vocMentionCount: number;
  vocMentionTrend: number;      // % change
  vocSentiment: number;         // -1 to 1
  vocSegmentWeight: number;     // Enterprise mentions weighted higher
  
  // Competitive factors
  competitorCount: number;      // How many competitors have this
  competitorRecency: number;    // How recently they launched
  marketExpectation: boolean;   // Is this table stakes?
  
  // Business factors
  blockedRevenue: number;       // $ in blocked deals
  churnRisk: number;            // $ at risk of churning
  expansionPotential: number;   // $ in expansion opportunity
  
  // Strategic factors
  strategicAlignment: number;   // 1-10 alignment with company goals
  technicalReadiness: number;   // 1-10 how ready we are to build
}

function calculatePriorityScore(factors: PriorityFactors): number {
  // Normalize each factor to 0-10 scale
  const vocScore = normalizeVoC(factors);
  const competitiveScore = normalizeCompetitive(factors);
  const businessScore = normalizeBusiness(factors);
  const strategicScore = normalizeStrategic(factors);
  
  // Weighted combination
  return (
    vocScore * 0.25 +
    competitiveScore * 0.20 +
    businessScore * 0.35 +
    strategicScore * 0.20
  );
}
```

### Component 4: Change Detection

Detect and alert on significant changes:

```typescript
async function detectPriorityChanges() {
  const items = await getRoadmapItems();
  const alerts = [];
  
  for (const item of items) {
    const currentScore = item.priorityScore;
    const previousScore = item.scoreHistory[item.scoreHistory.length - 1]?.score || currentScore;
    const scoreDelta = currentScore - previousScore;
    
    // Significant increase
    if (scoreDelta > 1.0) {
      alerts.push({
        type: 'priority_increase',
        item: item.id,
        from: previousScore,
        to: currentScore,
        reason: identifyChangeReason(item),
      });
    }
    
    // Significant decrease
    if (scoreDelta < -1.0) {
      alerts.push({
        type: 'priority_decrease',
        item: item.id,
        from: previousScore,
        to: currentScore,
        reason: identifyChangeReason(item),
      });
    }
    
    // New blocker
    if (item.evidence.blockedDeals.length > 0 && previousBlockedDeals === 0) {
      alerts.push({
        type: 'new_blocker',
        item: item.id,
        deals: item.evidence.blockedDeals,
      });
    }
  }
  
  return alerts;
}
```

## Automation with AI Agents

AI agents power the agentic roadmap:

### What Agents Do

- **Aggregate evidence** from all sources continuously
- **Calculate priority scores** based on current data
- **Detect changes** and generate alerts
- **Generate reports** for roadmap reviews
- **Suggest new items** based on emerging signals

### Sample Workflow

```typescript
// Weekly Roadmap Update Job

async function updateAgenticRoadmap() {
  // Step 1: Refresh evidence for all items
  const items = await getRoadmapItems();
  
  for (const item of items) {
    await refreshItemEvidence(item);
  }
  
  // Step 2: Recalculate priorities
  await recalculateAllPriorities();
  
  // Step 3: Detect changes
  const changes = await detectPriorityChanges();
  
  // Step 4: Identify new signals
  const newSignals = await identifyUnmappedSignals();
  
  // Step 5: Generate report
  const report = generateRoadmapReport({
    items,
    changes,
    newSignals,
  });
  
  return {
    type: 'roadmap_update',
    content: report,
    alerts: changes.filter(c => c.significance === 'high'),
  };
}
```

## FAQ

**Q: How do I transition from a static roadmap?**
A: Start by linking existing items to evidence. Add evidence connections gradually. The roadmap becomes "agentic" as connections grow.

**Q: What if stakeholders don't trust algorithmic prioritization?**
A: The algorithm suggests; humans decide. Use scores as input to discussion, not as final authority.

**Q: How do I handle strategic initiatives without customer evidence?**
A: Include strategic alignment as a factor. Some items are important for reasons beyond current customer feedback.

**Q: What about confidential roadmap items?**
A: Agentic roadmaps work with access controls. Evidence links can be restricted by role.

**Q: How often should priorities recalculate?**
A: Weekly for most teams. More frequent for fast-moving markets or during critical periods.

**Q: What if the algorithm keeps changing priorities?**
A: Add stability factors. Require sustained signal change before priority shifts. Avoid thrashing.
